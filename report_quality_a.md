# 코드 품질 평가 — Scenario A (구현 파일 기준)

도구: Ruff (린팅) + Radon CC/MI/Halstead (복잡도)
대상: `csv_stats.py` 구현 파일만 (테스트 파일 제외)

---

## 1. Ruff — 린팅

| 항목 | SDD | TDD (pi/Qwen3.5) |
|------|-----|-----------------|
| 오류 수 | **1** | **3** |
| 내용 | F401: `math` 미사용 import | E402: import 위치 위반 ×2, F401: `Callable` 미사용 import |
| 자동 수정 가능 | 1개 | 1개 |

**판정**: TDD 3 : SDD 1 — SDD 우위.
SDD의 `math` import는 단순 실수지만, TDD는 파일 중간에 import 추가(`E402`)라는 구조적 문제가 있음.

---

## 2. Radon CC — 순환 복잡도 (Cyclomatic Complexity)

낮을수록 단순하고 테스트하기 쉬움. A(<5) B(6-10) C(11-15) D(16-20)

| 함수 | SDD | TDD |
|------|-----|-----|
| `parse_csv` | A (5) | A (5) |
| `compute_stats` | **C (15)** | C (13) |
| `_detect_numeric_columns` | — | A (4) |
| `summarize` | A (1) | A (1) |
| **평균** | **B (7.0)** | **B (5.75)** |

**판정**: TDD 평균 복잡도 소폭 우위 (5.75 vs 7.0).
TDD는 `_detect_numeric_columns` 헬퍼로 책임을 분리해서 `compute_stats` CC가 낮아짐.
그러나 SDD는 함수 수가 적어 전체 구조는 더 단순함.

---

## 3. Radon MI — 유지보수성 지수 (Maintainability Index)

100에 가까울수록 유지보수 쉬움. A(≥20) B(10-19) C(<10)

| | SDD | TDD |
|--|-----|-----|
| MI 점수 | **56.75 (A)** | **51.03 (A)** |

**판정**: 둘 다 A등급. SDD가 5.72점 높음. 큰 차이는 아니지만 SDD가 소폭 더 유지보수하기 쉬운 코드.

---

## 4. Radon Halstead — 할스테드 복잡도

코드의 인지적 부담과 버그 가능성을 추정하는 지표.

| 지표 | SDD | TDD | 비율 |
|------|-----|-----|------|
| Volume (정보량) | 95.9 | 152.2 | TDD 1.59× 큼 |
| Difficulty (난이도) | 3.75 | 5.25 | TDD 1.40× 어려움 |
| Effort (구현 노력) | 359.7 | 798.8 | TDD **2.22×** 큼 |
| 예상 버그 수 | 0.032 | 0.051 | TDD 1.59× 많음 |

**판정**: SDD 압도적 우위. TDD(pi) 코드가 동일 기능 대비 인지 복잡도가 2배 이상 높고 버그 예상치도 60% 높음.

## 종합 요약

| 평가 항목 | SDD | TDD (pi) | 승자 |
|-----------|-----|----------|------|
| Ruff 오류 수 | 1 | 3 | **SDD** |
| CC 평균 | 7.0 | 5.75 | TDD (소폭) |
| MI 점수 | 56.75 | 51.03 | **SDD** |
| Halstead Effort | 359.7 | 798.8 | **SDD** |
| 예상 버그 수 | 0.032 | 0.051 | **SDD** |

**코드 품질은 SDD가 전반적으로 우위.**
CC에서만 TDD가 소폭 앞서나, 이는 함수를 쪼갠 덕분으로 설계 판단의 차이일 뿐.
Halstead 기준으로 TDD(pi) 코드는 SDD 대비 2배 이상의 인지적 부담을 가짐.

> **해석**: 토큰 절감(TDD 40% 저렴)과 코드 품질(SDD 우위) 사이에 트레이드오프 존재.
> pi/Qwen3.5가 기능은 구현하지만 코드 간결성·구조 면에서 Claude Sonnet에 못 미침.
